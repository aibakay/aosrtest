## Исправленные ошибки

| Ошибка | Решение |
|---|---|
| TS1117: дублирующиеся ключи в `FIELD_META` (`Наим_пр_докум`, `Название_работ`, `Должность_ФИО_Застройщика`) | Удалены вторые вхождения в `backend/src/config/templates.ts` |

| Закладка не находится при самозакрывающемся `<w:bookmarkEnd/>` | Regex обновлён на `<w:bookmarkEnd[^/]*/>` |

## Доработки (ревью + исправления)

| Что | Изменение |
|---|---|
| `xml:space="preserve"` ставился на любой экранированный спецсимвол и затирал атрибуты run'а | Логика исправлена: `preserve` только при ведущих/концевых пробелах, атрибуты run'а сохраняются (`bookmarkFiller.ts`) |
| Закладка из нескольких run'ов оставляла «хвосты» старого плейсхолдера | Значение пишется в первый `<w:t>`, остальные `<w:t>` очищаются |
| Файлы не сохранялись в `/backend/output` (требование CLAUDE.md) | Добавлено best-effort сохранение; на Vercel пишется в tmp (ФС read-only) |
| Отсутствовал глобальный error-handler / 404 | Вынесено в `app.ts`: 404 + централизованный error-handler, лимит JSON 1 МБ |
| `getTemplatePath` без защиты от path traversal | Принимаются только зарегистрированные коды шаблонов |
| Не было автотестов | `node:test`: 12 тестов на `fillBookmarks` и `splitDate` (`npm test`) |
| `node_modules` и `output/*.docx` были закоммичены | Добавлен `.gitignore`, файлы убраны из индекса |

## Ограничения (вне MVP)

- **PDF-экспорт** — не реализован. Требует MS Word / LibreOffice на сервере.
- **Реестр приложений (Power Query)** — в оригинале автозаполнялся из Excel; в веб-версии вводится вручную.
- **Копирование вложенных файлов** (схемы, протоколы, фото) — не реализовано.
- **Персистентность** — форма не сохраняет данные между сессиями браузера.
- **АОСР3 / АОСР_старый / АООК_старый** — работают (аналогичная структура), отдельно не тестировались.
