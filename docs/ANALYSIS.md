## 3. Механизм генерации (главное)

Генератор **не использует docxtemplater-плейсхолдеры**. Он работает через **Word Bookmarks (закладки)** и COM-автоматизацию Word:

1. `создать_акты_новые` перебирает строки таблицы `т_База_актов` (лист «База актов»).
2. Для каждой строки с флагом `Создать акт = "да"` определяется `Тип` (= имя шаблона).
3. `CreateWordDocument`: копирует `[Шаблоны]\<Тип>.docx` в выходную папку и открывает копию в Word.
4. Строится `Dictionary` соответствий **`имя_закладки → значение`** (`AddBasicActData`, `AddDateData`, `AddPeopleData` + специфика типа).
5. `FillBookmarks`: для каждого ключа словаря, если в документе есть закладка с таким именем — `bookmark.Range.Text = значение`.
6. `RemoveUnnecessaryRows`: если подписант (ГП/СКГП/АН/Подрядчик/Иное1-3) пуст — строка таблицы с его закладкой удаляется.
7. Сохранение `.docx` (+ опционально экспорт в PDF), копирование приложений.

**Вывод для веб-версии:** механизм = «заполнить именованные закладки docx значениями + удалить пустые строки подписантов». Это полностью воспроизводимо на Node без Word/COM (через прямую правку `word/document.xml`).

### Диспетчер типов (`CreateActByType`)
Маршрутизация по подстроке в имени типа:
`АОСР→mod_03`, `АООК→mod_04`, `АОГРОКС→mod_07`, `АРООКС→mod_06`, `АОУСИТО→mod_05`, иначе → `ЛюбойАкт` (modCustomActs).

## 4. Источники данных (Excel)

- **`т_База_актов`** (лист «База актов») — одна строка = один акт. Колонки: `id`, `Тип`, `Создать акт`, `Дата акта/начала/окончания`, `Автоназвание работ`, `Последующие работы`, `Доп.сведения`, `Кол-во экз.`, подписанты (`СК Заказчика`, `Генподрядчик`, `СК Генподрядчика`, `Подрядчик`, `Авторский надзор`, `Иное лицо 1-3`), приложения (`Исполнительные схемы`, `Материалы`, `Протоколы`), и пользовательские колонки `#Поле` (заполняют закладку с тем же именем).
- **`т_Люди`** — справочник подписантов: `id`, `ФИО`, `должность`, `Организация`, `Реквизиты организации`, `Идентификационный номер НРС`. `getFIO` → «Фамилия И.О.», `getPeople` → должность/ФИО/реквизиты.
- **Именованные диапазоны проекта** (шапка, общие для всех актов): `название_объекта`, `адрес_объекта`, `наименование_заказчика`+`реквизиты_заказчика`, `…_генподрядчика`, `…_проектировщика`, `…_подрядчика`.
- **Power Query `qВсеДанныеВАкт`** — агрегирует материалы/схемы/протоколы по `id` акта (для АОСР). Сложная Excel-логика; в вебе заменяется прямым вводом текста приложений.
- **Настройки** (`Range("НАСТР_…")`) — флаги: создавать PDF, создавать реестры, удалять пустых подписантов, группировать акты и т.д.

## 5. Номер акта
`GetActNumber` собирает номер из настраиваемых сегментов `мс1..мс12` (константные + значения из колонок). В веб-версии упрощается до прямого поля «Номер акта».

## 7. Требуемые поля фронтенда (сводно)

Общие для всех «строительных» актов (АОСР/АООК/АОУСИТО/АРООКС/АОГРОКС):
наименование_объекта, реквизиты_заказчика/генподрядчика/проектировщика, Номер_акта,
дата акта (день/месяц/год), подписанты (должность+ФИО для ТНЗ/Г/ТНГ/Пр/Пд/И1-И3),
Наименование_работ, Наим_пр_докум, Наименование_материалов, Исполнительные_схемы,
даты начала/окончания, выполнено_с, Наименование_последующих_работ, Доп_сведения, экз, приложения.

Простые акты (АГИ, Пролив, Промывка, ОтЭфД, АИИО, АВК, АПрОб) — свой компактный набор полей,
определяемый закладками конкретного шаблона (часто: город, дата, 3 подписанта, наименование объекта/работ, технические параметры).

## 8. Выводы для реализации
1. Один универсальный движок: docx + карта «поле формы → закладка».
2. Закладки извлекаются из шаблонов автоматически — список полей формы строится из них.
3. Удаление пустых строк подписантов — воспроизвести на уровне таблиц docx.
4. Сложные Excel-агрегации (реестры, Power Query, копирование вложений) — вне MVP; пользователь вводит текст приложений напрямую.
5. Оригинальные шаблоны не трогаем; при необходимости — рабочие копии.
