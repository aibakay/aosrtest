# TEMPLATES.md

Все Word-шаблоны из `[Шаблоны]` и их закладки (Word Bookmarks).
Извлечено из `word/document.xml` каждого `.docx` (исключены служебные `_GoBack`, `_Hlk…`, `RANGE!…`).

## Механизм заполнения
Каждый шаблон содержит именованные **закладки**. Движок ставит в `bookmark.Range.Text` значение
из словаря с тем же ключом. Имя закладки = ключ маппинга = поле формы.

Стратегия для веб-версии:
1. Парсим `<w:bookmarkStart w:name="...">` каждого шаблона → список полей.
2. Поле формы → закладка по совпадению имени (1:1).
3. Заполнение: вставляем текст между `bookmarkStart`/`bookmarkEnd` в `document.xml`, форматирование сохраняется.
4. Пустые строки подписантов (Должность_*/ФИО_*) можно удалять (опционально, как в VBA) — для MVP оставляем пустыми.

## Дубликаты/варианты
`АООК` ≡ `АООК_старый`; `АОСР` ≡ `АОСР3` ≡ `АОСР_старый` (одинаковый набор закладок, разная вёрстка).
В UI показываем основной вариант; «старые» — как альтернативный шаблон того же типа.

## Общие («сквозные») закладки строительных актов
`Наименование_объекта, реквизиты_заказчика, реквизиты_генподрядчика, реквизиты_проектировщика,
Номер_акта, день_а/месяц_а/год_а (дата акта), Должность_ТНЗ/Г/ТНГ/Пр/Пд + ФИО_ТНЗ/Г/ТНГ/Пр/Пд,
Наименование_работ, Наименование_последующих_работ, выполнено_с, Доп_сведения, экз`.
ТНЗ=тех.надзор заказчика, Г=генподрядчик, ТНГ=стройконтроль генподрядчика, Пр=авторский надзор (проектировщик), Пд=подрядчик, И1-И3=иные лица.

## Две схемы подписантов
- **Строительные акты** (АОСР/АООК/АОУСИТО/АРООКС/АОГРОКС): раздельные закладки `Должность_X` + `ФИО_X`.
- **Простые акты** (АГИ/АИИО/АПрОб/ОтЭфД/Пролив/Промывка): одна закладка `Должность_ФИО_<Роль>` (застройщик/генподрядчик/подрядчик/…).
- **АВК**: собственная схема (`ГП_должность`+`ГП_ФИО1`, `СК_*`, `Зак_*`, `Иное_*`).

Поэтому генератор полей формы должен опираться на фактический список закладок шаблона, а не на единый шаблон полей.

## Поля с днём/месяцем/годом
Даты в шаблонах разнесены на 3 закладки (день_X / месяц_X / год_X), причём **месяц — словом в родительном падеже** (января, февраля…). В форме — один date-picker, бэкенд раскладывает на 3 закладки.
